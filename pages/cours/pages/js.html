1.Introduction à JavaScript

1.1 Origines et histoire

JavaScript a été créé par Brendan Eich en 1995 alors qu'il travaillait chez Netscape Communications. À l'origine, le langage était connu sous le nom de Mocha, puis renommé LiveScript, avant de finalement adopter le nom JavaScript. La décision de ce nom était en grande partie marketing, pour capitaliser sur la popularité du langage Java à l'époque.

Il est important de noter que malgré leur nom similaire, Java et JavaScript sont deux langages très différents. JavaScript a été conçu pour être un langage de script pour le web, permettant d'ajouter de l'interactivité aux sites web.

En 1997, JavaScript a été standardisé par l'ECMA International, une association de normalisation pour les systèmes d'information et de communication. Depuis lors, le langage a évolué, avec l'introduction de nombreuses nouvelles fonctionnalités et l'émergence de plusieurs frameworks et bibliothèques pour faciliter le développement en JavaScript.

1.2 Pourquoi utiliser JavaScript ?

JavaScript est devenu un langage incontournable pour le développement web. Il est utilisé pour rendre les pages web interactives et dynamiques. Qu'il s'agisse de valider des formulaires, de créer des animations, ou de charger du contenu de manière asynchrone, JavaScript est à la base de presque toutes les interactions sur le web moderne.

Au-delà du navigateur, JavaScript a également trouvé sa place sur le serveur avec Node.js, un environnement d'exécution qui permet d'utiliser JavaScript pour créer des applications serveur. Avec JavaScript, vous pouvez donc créer des applications web complètes, du frontend au backend.

En outre, la popularité de JavaScript signifie qu'il y a une grande communauté de développeurs, une multitude de ressources pour apprendre et de nombreuses opportunités d'emploi

1.3 Environnement de développement

Pour commencer à coder en JavaScript, vous n'avez besoin que d'un navigateur web et d'un éditeur de texte. Le navigateur exécute votre code JavaScript, tandis que l'éditeur de texte vous permet de l'écrire. Cela peut être aussi simple que Notepad, mais des éditeurs spécialisés comme Visual Studio Code, Atom ou Sublime Text offrent de nombreuses fonctionnalités qui facilitent la vie du développeur.

Pour un développement plus avancé, vous voudrez peut-être configurer un environnement de développement plus robuste. Cela peut inclure un serveur local pour tester votre code, un débogueur pour vous aider à résoudre les problèmes, et peut-être un système de gestion de versions comme Git pour suivre vos modifications.

Pour le développement côté serveur avec Node.js, vous aurez également besoin d'installer Node.js et un gestionnaire de paquets comme npm ou yarn.

2.Syntaxe de base

2.1 Variables et types de données

Une variable est un conteneur pour stocker une valeur. En JavaScript, nous déclarons une variable en utilisant les mots-clés var, let ou const. Par exemple :

let name = 'John';
const age = 25;

Le choix entre var, let et const dépend de comment et où vous souhaitez utiliser la variable.

JavaScript est un langage à typage dynamique, ce qui signifie que le type de données d'une variable peut changer au fil du temps. Les types de données de base en JavaScript sont :

  • Number : pour les nombres, par exemple : const age = 25;
  • String : pour les textes, par exemple : let name = 'John';
  • Boolean : pour les valeurs vraies/fausses, par exemple : let isOnline = true;
  • Null : représente l'absence de valeur, par exemple : let something = null;
  • Undefined : signifie qu'une variable a été déclarée mais n'a pas encore été attribuée, par exemple : let test;
  • Object : pour les objets plus complexes, par exemple : let person = {firstName: 'John', lastName: 'Doe'};

2.2 Opérateurs

JavaScript comprend un certain nombre d'opérateurs qui permettent de manipuler des valeurs. Les principaux types d'opérateurs sont :

  • Opérateurs arithmétiques : comme +, -, *, /, % (modulo), ++ (incrément), -- (décrément)
  • Opérateurs d'affectation : comme =, +=, -=, *=, /=
  • Opérateurs de comparaison : comme == (égalité), != (non égalité), === (égalité stricte), !== (non égalité stricte), > (supérieur à), < (inférieur à), >= (supérieur ou égal), <= (inférieur ou égal)
  • Opérateurs logiques : comme && (et), || (ou), ! (non)

2.3 Structures de contrôle (conditions, boucles)

JavaScript offre plusieurs structures de contrôle pour aider à gérer le flux du programme.

  • Instructions conditionnelles : Elles permettent d'exécuter différents blocs de code en fonction de conditions spécifiques. Les structures de contrôle conditionnelles comprennent if, else if, else, et switch.

    if (age >= 18) {
    console.log('Vous pouvez voter.');
    } else {
    console.log('Vous ne pouvez pas voter.');
    }

  • Boucles : Elles permettent de répéter un bloc de code plusieurs fois. Les principales structures de boucles en JavaScript sont for, while, et do...while.

    for (let i = 0; i < 10; i++) {
    console.log(i);
    }

3.Fonctions

3.1 Définition et utilisation

Une fonction est un bloc de code conçu pour effectuer une tâche particulière. Une fonction est exécutée lorsque quelque chose l'invoque (l'appelle). Pour définir une fonction en JavaScript, on utilise le mot-clé function suivi du nom de la fonction, de parenthèses () qui peuvent contenir des paramètres et d'accolades {} qui contiennent le corps de la fonction. Voici un exemple :

function greet(name) {
console.log("Bonjour, " + name);
}

greet("Alice"); // Affiche : Bonjour, Alice

3.2 Portée des variables

La portée d'une variable désigne la partie du code où cette variable peut être référencée ou accédée.

  • Une variable déclarée à l'extérieur d'une fonction a une portée globale. Cela signifie qu'elle peut être utilisée n'importe où dans le code, y compris à l'intérieur des fonctions.
  • Une variable déclarée à l'intérieur d'une fonction a une portée locale. Cela signifie qu'elle ne peut être utilisée qu'à l'intérieur de cette fonction.
  • Les variables déclarées avec let ou const ont également une portée de bloc, ce qui signifie qu'elles ne sont accessibles qu'à l'intérieur du bloc de code où elles ont été déclarées.

let globalVar = "Je suis globale";
function myFunction() {
let localVar = "Je suis locale";
console.log(globalVar); // Peut accéder à la variable globale
console.log(localVar); // Peut accéder à la variable locale
}

myFunction();

console.log(globalVar); // Peut accéder à la variable globale
console.log(localVar); // Erreur : localVar n'est pas défini

3.3 Fonctions anonymes et fléchées

  • Une fonction anonyme est une fonction sans nom. Ces fonctions sont souvent utilisées comme arguments d'autres fonctions ou comme méthodes d'objets.

    let sayHello = function() {
    console.log("Bonjour !");
    };
    sayHello(); // Affiche : Bonjour !

  • Les fonctions fléchées sont une syntaxe plus courte pour définir une fonction. Elles ont été introduites avec ES6 (ES2015) et offrent quelques avantages par rapport aux fonctions traditionnelles, notamment une syntaxe plus concise et le fait qu'elles n'ont pas leur propre contexte this.

    let sayHello = () => {
    console.log("Bonjour !");
    };

    sayHello(); // Affiche : Bonjour !

4.Objets et Tableaux

4.1 Création et manipulation

  • Objets : Un objet est une collection de paires clé-valeur. Les clés sont appelées propriétés et peuvent référencer des valeurs de tout type, y compris d'autres objets ou des fonctions. Voici comment vous pouvez créer et manipuler un objet :

    let person = {
    firstName: 'John',
    lastName: 'Doe',
    age: 25
    };
    console.log(person.firstName); // Affiche : John
    // Ajouter une nouvelle propriété
    person.occupation = 'developer';
    console.log(person.occupation); // Affiche : developer

    // Modifier une propriété existante
    person.age = 26;
    console.log(person.age); // Affiche : 26

  • Tableaux : Un tableau est une liste ordonnée de valeurs. En JavaScript, les tableaux sont des objets et peuvent donc contenir des valeurs de tout type. Les indices de tableau commencent à 0.

    let fruits = ['apple', 'banana', 'cherry'];

    console.log(fruits[0]); // Affiche : apple

    // Ajouter un nouvel élément à la fin du tableau
    fruits.push('date');
    console.log(fruits); // Affiche : ['apple', 'banana', 'cherry', 'date']

    // Modifier un élément existant
    fruits[1] = 'blueberry';
    console.log(fruits); // Affiche : ['apple', 'blueberry', 'cherry', 'date']

4.2 Méthodes d'objets

En JavaScript, les fonctions peuvent être stockées en tant que valeurs de propriétés dans les objets. Lorsqu'une fonction est une propriété d'un objet, on l'appelle une méthode.

let person = {
firstName: 'John',
lastName: 'Doe',
fullName: function() {
return this.firstName + ' ' + this.lastName;
}
};

console.log(person.fullName()); // Affiche : John Doe

4.3 Les tableaux et leurs méthodes

JavaScript offre une large gamme de méthodes pour manipuler les tableaux :

  • push() : ajoute un ou plusieurs éléments à la fin d'un tableau et renvoie la nouvelle longueur du tableau.
  • pop() : supprime le dernier élément d'un tableau et le renvoie.
  • shift() : supprime le premier élément d'un tableau et le renvoie.
  • unshift() : ajoute un ou plusieurs éléments au début d'un tableau et renvoie la nouvelle longueur du tableau.
  • splice() : change le contenu d'un tableau en supprimant, remplaçant ou ajoutant des éléments.
  • slice() : renvoie une copie superficielle d'une portion de tableau.
  • sort() : trie les éléments d'un tableau en place et renvoie le tableau.

let fruits = ['apple', 'banana', 'cherry'];

fruits.push('date'); // ['apple', 'banana', 'cherry', 'date']
fruits.pop(); // ['apple', 'banana', 'cherry']
fruits.unshift('kiwi'); // ['kiwi', 'apple', 'banana', 'cherry']
fruits.shift(); // ['apple', 'banana', 'cherry']
fruits.sort(); // ['apple', 'banana', 'cherry']

let slicedFruits = fruits.slice(1, 3); // ['banana', 'cherry']

5.JavaScript et le navigateur web

5.1 Le Document Object Model (DOM)

Le Document Object Model (DOM) est une interface de programmation pour les documents HTML et XML. Il représente la structure d'un document web sous forme d'arbre, ce qui permet aux langages de programmation comme JavaScript de manipuler le contenu, la structure et le style d'un site web.

Dans le DOM, chaque élément HTML est représenté par un objet avec des propriétés et des méthodes. Par exemple, dans le DOM, la balise <p> est un objet qui a des propriétés comme innerHTML et des méthodes comme setAttribute().

5.2 Manipulation du DOM

JavaScript peut manipuler le DOM de plusieurs façons, notamment en ajoutant, modifiant ou supprimant des éléments et des attributs HTML. Voici quelques exemples :

  • Sélection d'éléments : Vous pouvez sélectionner des éléments HTML en utilisant document.getElementById(), document.getElementsByClassName(), document.getElementsByTagName(), ou document.querySelector().
  • Modification de contenu : Vous pouvez modifier le contenu d'un élément HTML en utilisant la propriété innerHTML. Par exemple, document.getElementById('myId').innerHTML = 'New content';.
  • Modification d'attributs : Vous pouvez modifier les attributs d'un élément HTML en utilisant setAttribute(). Par exemple, document.getElementById('myId').setAttribute('class', 'myClass');.

5.3 Gestion des événements

Un événement est une action qui se produit dans le navigateur. Il peut être déclenché par l'utilisateur (un clic de souris, une frappe de touche, un mouvement de souris, etc.) ou par le navigateur lui-même (la fin du chargement d'une page, un minuteur qui se déclenche, etc.).

JavaScript permet de définir des fonctions appelées gestionnaires d'événements qui s'exécutent lorsqu'un événement spécifique se produit. Voici un exemple de gestionnaire d'événements :

let button = document.getElementById('myButton');
button.addEventListener('click', function() {
console.log('Le bouton a été cliqué !');
});

Dans cet exemple, la fonction est exécutée chaque fois que l'utilisateur clique sur le bouton avec l'ID 'myButton'.

6.JavaScript avancé

6.1 Fonctions de rappel (callback) et Promesses

  • Fonctions de rappel (Callback) : Une fonction de rappel est une fonction qui est passée en tant qu'argument à une autre fonction, puis invoquée par cette dernière pour accomplir une sorte de routine ou d'action.

    function greet(name, callback) {
    console.log('Bonjour, ' + name);
    callback();
    }

    function alertEnd() {
    console.log('Fin de la conversation.');
    }

    greet('Alice', alertEnd);

  • Promesses : Une promesse est un objet qui peut renvoyer une valeur dans le futur. Les promesses sont utilisées pour gérer des opérations asynchrones en JavaScript, comme l'exécution d'une requête AJAX. Une promesse peut être dans l'un des trois états : en attente (pending), résolue (fulfilled), ou rejetée (rejected).

    let promise = new Promise((resolve, reject) => {
    let operation = true; // Simule une opération réussie

    if(operation) {
    resolve('Operation réussie');
    } else {
    reject('Operation échouée');
    }
    });
    promise
    .then(message => console.log(message)) // 'Operation réussie'
    .catch(error => console.log(error));

6.2 Asynchronous JavaScript (async/await)

async et await sont des extensions de Promises et permettent d'écrire du code asynchrone de manière plus facile à lire et à comprendre. Une fonction async renvoie une Promesse. Le mot-clé await peut seulement être utilisé à l'intérieur d'une fonction async, et fait que l'exécution du code asynchrone se comporte comme si elle était synchrone.

async function asyncFunction() {
let response = await fetch('https://api.example.com/data');
let data = await response.json();

console.log(data);
}

asyncFunction();

6.3 Closures et IIFE

  • Closures : Un closure est une fonction qui a accès à son propre scope, au scope de la fonction englobante et au scope global. En d'autres termes, un closure est une fonction à l'intérieur d'une autre fonction et a accès aux variables de la fonction parente.

    function outerFunction(outerVariable) {
    return function innerFunction(innerVariable) {
    console.log('outerVariable:', outerVariable);
    console.log('innerVariable:', innerVariable);
    }
    }

    let newFunction = outerFunction('outside');
    newFunction('inside'); // Logs: 'outerVariable: outside' and 'innerVariable: inside'

  • IIFE : Une IIFE (Immediately Invoked Function Expression) est une fonction en JavaScript qui est exécutée dès qu'elle est définie. Les IIFE sont souvent utilisées pour éviter d'exposer des variables à la portée globale.

    (function() {
    let temp = 'Variable temporaire';
    console.log(temp);
    })(); // Logs: 'Variable temporaire'

7.Programmation orientée objet en JavaScript

7.1 Classes et héritage

JavaScript est un langage orienté objet qui utilise le prototypage plutôt que les classes dans sa conception originale. Cependant, à partir de ES6, JavaScript a introduit le mot-clé class pour faciliter la création d'objets et l'héritage.

  • Classes : Une classe est une sorte de modèle pour la création d'objets. Elle encapsule les données et les fonctions qui manipulent ces données.

    class Animal {
    constructor(name) {
    this.name = name;
    }

    speak() {
    console.log(this.name + ' fait un bruit.');
    }
    }

    let animal = new Animal('Un animal');
    animal.speak(); // Affiche : 'Un animal fait un bruit.'

  • Héritage : L'héritage est un principe de la programmation orientée objet qui permet de créer une nouvelle classe à partir d'une classe existante.

    class Dog extends Animal {
    speak() {
    console.log(this.name + ' aboie.');
    }
    }

    let dog = new Dog('Rex');
    dog.speak(); // Affiche : 'Rex aboie.'

7.2 Prototypes

Dans JavaScript, chaque objet a une propriété spéciale appelée prototype. Le prototype est aussi un objet, et tous les objets héritent de propriétés et de méthodes de leur prototype.

  • Héritage de prototype : Lorsque vous essayez d'accéder à une propriété ou à une méthode d'un objet, JavaScript cherche d'abord cette propriété ou méthode sur l'objet lui-même. Si elle ne la trouve pas, elle la cherche sur le prototype de l'objet, et ainsi de suite, jusqu'à ce qu'elle trouve la propriété ou qu'elle atteigne la fin de la chaîne de prototypes.

    let animal = {
    speaks: true
    };

    let dog = Object.create(animal);
    console.log(dog.speaks); // Affiche : true

7.3 ES6 et au-delà

ES6, ou ECMAScript 2015, a apporté de nombreux changements à JavaScript, notamment les classes, les modules, les promesses, les flèches et bien d'autres fonctionnalités. Depuis ES6, ECMAScript publie de nouvelles spécifications chaque année, ajoutant de nouvelles fonctionnalités à JavaScript. Voici quelques exemples :

  • ES7 (2016) : comprend les opérateurs exponentiels et Array.prototype.includes.
  • ES8 (2017) : comprend async/await, Object.values(), Object.entries() et String.prototype.padStart()/padEnd().
  • ES9 (2018) : comprend le Rest/Spread Properties, Promise.finally(), et RegExp improvements.
  • ES10 (2019) : comprend Array.prototype.flat(), Array.prototype.flatMap(), Object.fromEntries(), String.prototype.trimStart()/trimEnd(), et try/catch binding.

8.Gestion des erreurs et débogage

8.1 Les erreurs communes

Dans le développement JavaScript, certaines erreurs sont plus communes que d'autres. Voici quelques exemples :

  • Référence non définie : Cette erreur se produit lorsque vous essayez d'accéder à une variable qui n'a pas été déclarée. Par exemple : console.log(inexistentVar);.
  • TypeError : Cette erreur se produit lorsque vous essayez d'effectuer une opération non valide sur un certain type de données. Par exemple : undefined.split(' ');.
  • Erreur de syntaxe : Cette erreur se produit lorsque JavaScript ne comprend pas la syntaxe du code. Par exemple : const name = 'Alice'';.

8.2 Utilisation des outils de débogage

Les outils de débogage sont essentiels pour comprendre et résoudre les erreurs dans votre code. Voici quelques outils de débogage en JavaScript :

  • Console de navigateur : Les consoles des navigateurs web modernes, comme Chrome DevTools, Firefox Developer Tools, ou Safari Web Inspector, offrent des outils de débogage puissants. Vous pouvez utiliser console.log(), console.error(), et d'autres méthodes de la console pour afficher des messages de débogage.
  • Breakpoints : Vous pouvez utiliser des points d'arrêt pour arrêter l'exécution du code à un certain point et inspecter les valeurs des variables à ce moment-là.
  • Fonction debugger : La fonction debugger; de JavaScript arrête l'exécution du code et déclenche l'outil de débogage du navigateur, si celui-ci est ouvert.

8.3 Bonnes pratiques pour éviter les erreurs

  • Utilisez 'use strict'; : L'instruction 'use strict'; en haut de votre script force JavaScript à être plus strict sur les erreurs, ce qui peut vous aider à les éviter.
  • Revoyez votre code : Passer en revue votre code peut vous aider à détecter les erreurs avant qu'elles ne se produisent. Recherchez les variables non déclarées, les typos, les erreurs de syntaxe, etc.
  • Utilisez des linters : Les linters comme ESLint peuvent analyser votre code pour détecter des erreurs potentielles et des problèmes de style.
  • Testez votre code : Écrivez des tests pour votre code pour vérifier que tout fonctionne comme prévu. Vous pouvez utiliser des cadres de test comme Jest ou Mocha pour cela.

9.Utilisation de bibliothèques et de frameworks JavaScript

9.1 Introduction à jQuery

jQuery est une bibliothèque JavaScript rapide et concise qui simplifie la manipulation du document HTML, la gestion des événements, l'animation et l'interaction Ajax. C'est une excellente bibliothèque pour commencer à améliorer l'interaction utilisateur sans la complexité d'autres frameworks plus lourds.

9.2 Introduction à React.js

React.js est une bibliothèque JavaScript pour la construction d'interfaces utilisateur, développée par Facebook. React permet aux développeurs de créer de grandes applications web qui peuvent changer les données, sans recharger la page. L'objectif principal de React est d'être rapide, scalable, et simple. Il fonctionne uniquement sur les interfaces utilisateur dans l'application.

9.3 Introduction à Node.js

Node.js est une plateforme logicielle open-source qui utilise le moteur JavaScript V8 de Chrome pour interpréter le code JavaScript côté serveur. Node.js est principalement utilisé pour créer des applications de réseau, comme des serveurs web, en raison de sa capacité à gérer un grand nombre de connexions simultanées avec une grande performance.

10.Tests et déploiement

10.1 Écriture de tests unitaires avec Jest

Jest est un cadre de test JavaScript populaire qui permet d'écrire des tests unitaires pour votre code. Les tests unitaires vérifient que les parties individuelles (ou unités) de votre code fonctionnent comme prévu.

// myFunctions.js
function add(a, b) {
return a + b;
}

module.exports = add;

// myFunctions.test.js
const add = require('./myFunctions');

test('add 1 + 2 pour obtenir 3', () => {
expect(add(1, 2)).toBe(3);
});

Pour exécuter les tests, vous pouvez utiliser la commande npm test ou yarn test dans votre terminal.

10.2 Automatisation des tests

L'automatisation des tests permet de s'assurer que votre code fonctionne correctement après chaque modification. Des outils comme Jenkins, Travis CI, ou GitHub Actions peuvent être utilisés pour exécuter automatiquement vos tests chaque fois que vous soumettez des modifications à votre code.

10.3 Déploiement sur un serveur

Le déploiement est le processus de mise en ligne de votre application pour qu'elle soit accessible aux utilisateurs. Le processus de déploiement peut varier en fonction de l'infrastructure que vous utilisez. Voici un exemple de processus de déploiement simple avec Node.js et Heroku :

  1. Créez une application sur Heroku.
  2. Installez le CLI Heroku sur votre machine.
  3. Connectez-vous à Heroku avec heroku login.
  4. Initialiser un dépôt Git dans votre projet avec git init.
  5. Ajoutez tous vos fichiers à Git avec git add . et commitez vos modifications avec git commit -m "Message de commit".
  6. Connectez votre dépôt local à Heroku avec heroku git:remote -a nom-de-votre-application.
  7. Déployez votre application sur Heroku avec git push heroku master.

En écrivant des tests pour votre code et en automatisant ces tests, vous pouvez améliorer la qualité de votre code et détecter rapidement les erreurs. Et en apprenant à déployer votre application sur un serveur, vous pouvez rendre votre application accessible aux utilisateurs du monde entier.

11.Projets pratiques

11.1 Application de gestion de tâches (ToDo)

Une application de gestion des tâches est un bon projet pour commencer à pratiquer vos compétences en JavaScript. Elle peut inclure des fonctionnalités comme ajouter des tâches, les marquer comme terminées et les supprimer.

11.2 Mini-jeu en JavaScript

Un mini-jeu peut être un excellent moyen de mettre en pratique et d'améliorer vos compétences en JavaScript. Cela pourrait être aussi simple qu'un jeu de casse-briques, un jeu de serpent, ou un jeu de correspondance de cartes.

11.3 Application CRUD avec Node.js

Un projet CRUD (Create, Read, Update, Delete) est un autre excellent moyen de pratiquer vos compétences en JavaScript et Node.js. Cela pourrait être une application de blog, une application de gestion des employés, etc.

12.Conclusion et perspectives

12.1 Conseils pour continuer à apprendre

Apprendre à programmer avec JavaScript est une aventure continue. Voici quelques conseils pour continuer à apprendre :

  • Pratiquez régulièrement : La pratique est la clé pour devenir un bon développeur. Continuez à coder, à résoudre des problèmes et à travailler sur des projets.
  • Rejoignez la communauté : Il existe de nombreuses communautés en ligne où vous pouvez poser des questions, partager vos connaissances et apprendre des autres.
  • Apprenez d'autres technologies : Après avoir maîtrisé JavaScript, vous pouvez apprendre d'autres technologies comme TypeScript, Vue.js, Angular, etc.

12.2 Veille technologique

La veille technologique est essentielle dans le domaine du développement web en raison de sa nature en constante évolution. Suivez les blogs, les podcasts, les conférences et les forums dédiés à JavaScript et au développement web pour rester à jour.

13.Développement de carrière en JavaScript

JavaScript offre de nombreuses opportunités de carrière. Voici quelques chemins que vous pourriez envisager :

  • Développeur front-end : Spécialisez-vous dans la création d'interfaces utilisateur interactives avec JavaScript et des frameworks comme React ou Vue.js.
  • Développeur back-end : Utilisez Node.js pour créer des serveurs et des API.
  • Développeur full-stack : Combinez le développement front-end et back-end pour créer des applications complètes.
  • Développeur de jeux : Utilisez JavaScript pour créer des jeux pour le web ou pour les plateformes mobiles.
