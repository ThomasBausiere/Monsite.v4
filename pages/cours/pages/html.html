Cours HTML

1. Introduction au HTML

1.1. Qu'est-ce que le HTML ?

HTML, ou HyperText Markup Language, est un langage de balisage utilisé pour structurer le contenu sur le Web. Il définit la signification et la structure du contenu web. Les autres technologies à côté du HTML sont généralement utilisées pour décrire l'apparence/présentation (CSS) ou la fonctionnalité/comportement (JavaScript) d'une page Web.

1.2. Historique et évolution du HTML

HTML a été créé par Tim Berners-Lee en 1991 mais "HTML 2.0" a été la première norme HTML à être publiée en 1995. HTML 4.01 a été une grande version de HTML et elle a été publiée en 1999. L'HTML5 est la dernière version, publiée en octobre 2014 par le W3C.

1.3. Rôle du HTML dans le développement web

HTML joue un rôle clé dans toute application ou site web sur Internet. C'est le squelette de la page, qui donne la structure et place le contenu. Sans HTML, un navigateur web ne saurait pas comment structurer le contenu d'une page web.

2. Bases du HTML

2.1. Structure de base d'un document HTML

Un document HTML de base a une structure spécifique qui comprend les éléments suivants:

  • <!DOCTYPE html> : Cette déclaration doit être la première chose dans votre document HTML. Elle est utilisée pour indiquer au navigateur la version de HTML utilisée, dans ce cas HTML5.
  • <html> : C'est l'élément racine d'une page web.
  • <head> : Cet élément contient des méta-informations sur le document HTML qui n'est généralement pas visible pour les utilisateurs.
  • <title> : Cet élément définit le titre du document, qui est affiché dans la barre de titre du navigateur ou l'onglet de la page.
  • <body> : L'élément <body> contient le contenu visible d'une page web.

2.2. Balises HTML courantes et leurs utilisations

Il existe de nombreuses balises HTML, mais voici quelques-unes des plus courantes :

  • <h1> à <h6> : Ces balises sont utilisées pour définir les titres et sous-titres. <h1> est le titre le plus important, et < h6> est le moins important.
  • <p> : Cette balise est utilisée pour définir un paragraphe.
  • <a> : La balise <a> est utilisée pour créer des liens.
  • <img> : Cette balise est utilisée pour insérer des images.
  • <ul> et <ol> : Ces balises sont utilisées pour créer des listes. <ul> pour les listes non ordonnées et <ol> pour les listes ordonnées.
  • <li> : Cette balise est utilisée pour définir un élément de liste.

2.3. Commentaires HTML

Les commentaires en HTML commencent par <!-- et se terminent par -->. Ils sont utilisés pour ajouter des notes ou des descriptions dans le code HTML qui ne seront pas rendues par le navigateur.

<!-- Ceci est un commentaire -->

2.4. Les attributs HTML

Les balises HTML peuvent également avoir des attributs. Les attributs fournissent des informations supplémentaires sur les éléments. Ils sont toujours spécifiés dans la balise de départ et viennent généralement par paires sous la forme nom/valeur. Par exemple, l'attribut href pour la balise < a> est utilisé pour spécifier l'URL de la page vers laquelle le lien doit pointer.

<a href="https://www.exemple.com">Ceci est un lien</a>

Ceci est un lien

Voici un autre exemple avec l'attribut src (source) pour la balise <img> qui est utilisée pour spécifier le chemin de l'image.

<img src="image.jpg">

3.3. Balises de formatage de texte

HTML fournit de nombreuses balises pour formater le texte. Voici quelques exemples :

  • <b> pour le texte en gras
  • <i> pour le texte en italique
  • <u> pour le texte souligné
  • <s> pour le texte barré
  • <sup> pour le texte en exposant
  • <sub> pour le texte en indice
  • <pre> pour le texte préformaté
  • <code> pour le texte de code informatique
  • <blockquote> pour une citation longue

3.4. Listes HTML

Il existe trois types de listes en HTML :

  • Listes non ordonnées (<ul>), où chaque élément de la liste (<li>) est marqué d'une puce.
  • Listes ordonnées (<ol>), où chaque élément de la liste est numéroté.
  • Listes de définition (<dl>, <dt>, <dd>), où chaque élément de la liste est composé d'un terme et d'une définition.

4. Liens et images en HTML

4.1. Création de liens

En HTML, les liens sont définis par la balise <a>. L'attribut href est utilisé pour indiquer la destination du lien.

<a href="https://www.example.com">Ceci est un lien</a>

4.2. Insertion d'images

La balise <img> est utilisée pour insérer une image dans une page HTML. L'attribut src (source) est utilisé pour spécifier le chemin de l'image. Notez que la balise <img> est une balise vide, ce qui signifie qu'elle n'a pas de balise de fermeture.

<img src="chemin/vers/image.jpg" alt="Description de l'image">

L'attribut alt (alternative) est utilisé pour fournir un texte alternatif pour l'image, au cas où elle ne pourrait pas être affichée. Ce texte aide également à améliorer l'accessibilité pour les personnes qui utilisent des lecteurs d'écran.

4.3. Attributs de liens et d'images

Il existe plusieurs autres attributs que vous pouvez utiliser pour les balises <a> et <img>.
Pour les liens:

  • target="_blank" : ouvre le lien dans un nouvel onglet ou une nouvelle fenêtre.
  • title : affiche une info-bulle lorsque le curseur est placé sur le lien.

Pour les images:

  • width et height : spécifient la largeur et la hauteur de l'image.
  • title : affiche une info-bulle lorsque le curseur est placé sur l'image.

<a href="https://www.example.com" target="_blank" title="Cliquez ici pour visiter example.com">Ceci est un lien</a>

Ceci est un lien

<img src="chemin/vers/image.jpg" alt="Description de l'image" width="500" height="600" title="Ceci est une image">

5. Tables, Formulaires et iFrames en HTML

5.1. Création de tables

Les tables en HTML sont définies à l'aide des balises <table>, <tr> (table row), <th> (table header) et <td> (table data).

En-tête 1 En-tête 2
Donnée 1 Donnée 2

<table>
<tr>
<th>En-tête 1</th>
<th>En-tête 2</th>
</tr>
<tr>
<td>Donnée 1</td>
<td>Donnée 2</td>
</tr>
</table>

5.2. Création de formulaires

Les formulaires en HTML sont définis à l'aide de la balise <form>. Un formulaire peut contenir différents types d'éléments de saisie, tels que des champs de texte, des cases à cocher, des boutons radio, des boutons, etc.



<form>
<label for="name">Nom:</label>
<input type="text" id="name" name="name">
<input type="submit" value="Submit">
</form>

5.3. Utilisation des iFrames

Une iFrame (inline frame) est utilisée pour intégrer une autre page HTML dans une page HTML. Les iFrames sont définies à l'aide de la balise <iframe>.

<iframe src="https://www.example.com" title="Description de l'iFrame"></iframe>

L'attribut src spécifie l'URL de la page à intégrer. L'attribut title fournit un titre pour l'iFrame pour les technologies d'assistance, telles que les lecteurs d'écran.

6. Intégration du CSS et du JavaScript dans le HTML

6.1. Comment lier une feuille de style CSS à une page HTML

Pour appliquer des styles CSS à votre page HTML, vous pouvez utiliser une approche externe en créant un fichier séparé avec l'extension .css, puis en le liant à votre fichier HTML.

Dans votre fichier HTML, vous pouvez lier le fichier CSS en utilisant la balise <link> dans la section <head>.

Notez que href dans la balise <link> doit avoir le chemin du fichier CSS que vous avez créé.

6.2. Comment intégrer du code JavaScript dans une page HTML

Il existe deux façons principales d'intégrer du code JavaScript dans votre page HTML :

  1. Inline JavaScript :

    Vous pouvez écrire du code JavaScript directement dans votre document HTML en utilisant la balise <script> . Le code JavaScript peut être placé n'importe où, mais il est généralement placé dans la section < head> ou < body>.

    <button onclick=" alert('Bonjour, monde!')">Click me</button>

  2. External JavaScript :

    Similaire à la méthode CSS externe, vous pouvez écrire votre code JavaScript dans un fichier séparé (avec l'extension .js), puis le lier à votre fichier HTML. Pour ce faire, vous utilisez également la balise <'script>, mais cette fois avec l'attribut src qui contient le chemin vers le fichier JavaScript.

    <script src="script.js"> </script>

7. HTML5 et ses nouveautés

HTML5 est la dernière version de HTML, elle offre de nouvelles fonctionnalités, de nouvelles balises et améliore la facilité d'utilisation et l'interactivité de l'expérience web.

7.1. Nouvelles balises en HTML5

HTML5 a introduit de nombreuses nouvelles balises pour améliorer la structure et la sémantique des documents web. Parmi celles-ci :

  • <header>, <footer>, <article>, <section>, <aside> et <nav> pour une meilleure structure de page.
  • <figure>, <figcaption> pour les illustrations et leurs légendes.
  • <time> pour les dates et heures.
  • <main> pour le contenu principal unique d'un document.
  • <mark> pour le texte souligné.

7.2. Formulaires HTML5 et nouveaux types de champs

HTML5 a également introduit de nouvelles façons de gérer les formulaires et de nouveaux types de champs pour faciliter l'entrée de données. Par exemple :

  • Types de champs comme number, date, time, email, url, search, range et color.
  • Attributs de formulaire tels que placeholder, required, autofocus, autocomplete et pattern.

7.3. APIs HTML5 (Audio, Video, Canvas, etc.)

HTML5 a introduit plusieurs nouvelles API pour enrichir les fonctionnalités web :

  • API de vidéo et d'audio HTML5 : Avec les balises <video> et <audio>, il est désormais facile d'intégrer des médias directement dans vos documents web sans nécessiter de plugin externe.
  • API Canvas : La balise <canvas> permet de dessiner des graphiques dynamiques et des images bitmap directement dans le navigateur.
  • API de géolocalisation : Cette API permet aux applications web d'accéder à la position géographique de l'utilisateur, à condition que l'utilisateur donne son accord.
  • API de stockage web : HTML5 introduit le stockage local qui permet de stocker des données de manière plus sécurisée et efficace que les cookies.

Il y a beaucoup plus de nouvelles fonctionnalités et API introduites avec HTML5. Nous allons explorer ces sujets plus en détail dans les leçons suivantes.

8 Bonnes pratiques en HTML

Écrire du code HTML propre et maintenable est essentiel pour tout développeur web. Voici quelques bonnes pratiques pour vous aider à le faire.

8.1. Organisation et indentation du code

Une bonne organisation du code est essentielle pour la lisibilité et la maintenabilité. Chaque élément et chaque bloc de texte doivent être correctement indentés pour montrer clairement la structure de votre document. Cela peut être fait en utilisant une indentation de deux ou quatre espaces, ou une tabulation. Il est également utile d'utiliser des commentaires pour expliquer les parties complexes du code.

8.2. Accessibilité

L'accessibilité est une autre considération importante lors de l'écriture du HTML. Cela signifie que votre site web doit être utilisable par tous, y compris les personnes qui utilisent des technologies d'assistance comme les lecteurs d'écran. Certaines choses à garder à l'esprit comprennent :

  • Utiliser des balises sémantiques pour décrire le contenu de la page.
  • Fournir du texte alternatif pour les images avec l'attribut alt.
  • Assurez-vous que tous les éléments interactifs peuvent être utilisés avec le clavier.

8.3. SEO et HTML

Le SEO (Search Engine Optimization) est l'art d'optimiser un site web pour les moteurs de recherche. Certaines bonnes pratiques de SEO en HTML incluent :

  • Utiliser des balises de titre (<title>) et des balises de meta description (<meta name="description">) pour chaque page.
  • Utiliser des balises d'en-tête (<h1>, <h2>, etc.) pour structurer le contenu.
  • Utiliser l'attribut alt pour fournir des descriptions textuelles des images.
  • Fournir un sitemap pour aider les moteurs de recherche à indexer votre site.

En respectant ces bonnes pratiques, vous pouvez assurer que votre code HTML est bien structuré, accessible, et optimisé pour les moteurs de recherche.
