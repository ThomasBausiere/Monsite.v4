Cours de PHP

Partie 1: Introduction à PHP

1.1. Historique de PHP

PHP, qui signifie "PHP: Hypertext Preprocessor", est un langage de programmation de scripts côté serveur créé en 1995 par Rasmus Lerdorf. À l'origine, PHP était un ensemble simple de scripts pour suivre les visites de son CV en ligne. Au fil des ans, il a évolué pour devenir un langage de programmation à part entière, principalement utilisé pour les applications web. Les différentes versions de PHP ont apporté de nombreuses améliorations et fonctionnalités, faisant de PHP l'un des langages de programmation web les plus populaires au monde.

1.2. Pourquoi choisir PHP?

Plusieurs raisons font de PHP un choix judicieux pour le développement web :

  • Open-source : PHP est libre et gratuit à utiliser. Cela a encouragé une grande communauté de développeurs à contribuer, offrant une vaste gamme de ressources, de bibliothèques et de cadres.
  • Facilité d'utilisation : PHP est largement reconnu pour sa syntaxe simple et sa courbe d'apprentissage douce, ce qui le rend accessible aux nouveaux développeurs tout en étant suffisamment puissant pour les experts.
  • Intégration : PHP s'intègre bien avec une variété de bases de données et de systèmes, y compris MySQL, PostgreSQL, Oracle, et bien d'autres.
  • Flexibilité : Il peut être utilisé pour créer des sites web simples ou des applications web complexes.
  • Performance : Avec les améliorations continues apportées aux versions successives, PHP est devenu très performant et répond bien aux exigences des applications modernes.

1.3. Fonctionnement général de PHP dans le contexte web

PHP fonctionne côté serveur. Lorsqu'un utilisateur accède à une page web écrite en PHP, voici ce qui se passe :

  1. La demande de page est envoyée au serveur web.
  2. Le serveur reconnaît le script PHP en raison de son extension .php et passe la demande au processeur PHP.
  3. PHP exécute le script et génère du HTML en sortie.
  4. Le serveur renvoie ce HTML à la demande du navigateur.
  5. Le navigateur affiche le contenu HTML reçu, sans jamais connaître ou voir le code PHP sous-jacent.

Cette nature côté serveur de PHP signifie qu'il peut interagir avec des bases de données, lire et écrire des fichiers sur le serveur, et traiter les données de l'utilisateur avant d'envoyer une réponse, offrant ainsi une grande flexibilité et puissance pour les applications web.

Partie 2: Configuration et environnement de développement

2.1. Installation de PHP (Windows, Linux, MacOS)

L'installation de PHP varie en fonction du système d'exploitation. Voici un aperçu général pour les trois principaux systèmes d'exploitation :

  • Windows : Pour Windows, il existe des paquets tout-en-un, comme XAMPP ou WAMP, qui incluent PHP, Apache et MySQL. Sinon, vous pouvez télécharger PHP depuis le site officiel et le configurer manuellement.
  • Linux : La plupart des distributions Linux offrent PHP via leurs gestionnaires de paquets, tels que apt pour Debian/Ubuntu ou yum pour CentOS. Exemple de commande pour installer PHP sur Debian/Ubuntu :

    sudo apt-get update
    sudo apt-get install php

  • MacOS : Tout comme Linux, MacOS possède également des gestionnaires de paquets, comme Homebrew. Une fois Homebrew installé, PHP peut être installé avec la commande suivante:

    brew install php

2.2. Configuration d'un serveur local (comme Apache)

Un serveur local est essentiel pour exécuter des scripts PHP. Apache est l'un des serveurs web les plus populaires utilisés avec PHP.

  • Installation d'Apache : La méthode d'installation varie selon le système d'exploitation. Sur Debian/Ubuntu, cela peut être fait avec:

    sudo apt-get install apache2

  • Configuration : Une fois Apache installé, vous devrez le configurer pour exécuter PHP. Vous devez vous assurer que le module PHP est chargé dans la configuration d'Apache et définir la configuration pour traiter les fichiers `.php`.
  • Démarrage du serveur : Après la configuration, vous pouvez démarrer ou redémarrer le serveur Apache pour que les modifications prennent effet.

2.3. Introduction à php.ini

Le fichier `php.ini` est le fichier de configuration par défaut pour PHP. Il est utilisé pour contrôler les variables telles que les tailles de téléchargement de fichiers, les paramètres d'affichage d'erreur, et les extensions à charger.

  • Localisation : Le fichier `php.ini` se trouve généralement dans le répertoire d'installation de PHP, mais cela peut varier selon votre système et la manière dont PHP a été installé.
  • Modifications : Les modifications apportées à ce fichier nécessitent généralement un redémarrage du serveur web pour prendre effet.
  • Options courantes : Vous pouvez ajuster de nombreuses options dans `php.ini`, comme `memory_limit` pour contrôler la quantité de mémoire qu'un script PHP peut consommer, ou `display_errors` pour contrôler si les erreurs PHP sont affichées dans le navigateur ou non.

Il est essentiel de comprendre et de maîtriser `php.ini` lors de la configuration de votre environnement de développement et de production, car ce fichier a une influence significative sur le comportement de PHP.

Partie 3: Syntaxe de base

3.1. Structure de PHP

PHP est intégré au sein de documents HTML, ce qui le rend flexible pour la création de pages web dynamiques. Voici comment il est structuré :

<html>
<head>
<title>Exemple PHP</title>
</head>
<body>
<?php
echo "Bonjour, monde!";
?>
</body>
</html>

Dans cet exemple, PHP est utilisé à l'intérieur des balises ``, et la commande `echo` est utilisée pour afficher le texte sur la page web.

3.2. Variables et types de données

Les variables en PHP commencent par un signe dollar `$` suivi du nom de la variable. PHP est un langage faiblement typé, ce qui signifie que vous n'avez pas besoin de déclarer le type de données d'une variable lors de sa création.

  • Variables :

    $txt = "PHP";
    $x = 5;
    $y = 10.5;

  • Types de données : PHP supporte plusieurs types de données, notamment:
    • String (chaîne de caractères)
    • Integer (entier)
    • Float (nombre à virgule flottante)
    • Boolean (booléen)
    • Array (tableau)
    • Object (objet)
    • NULL

3.3. Opérateurs

Les opérateurs sont utilisés pour effectuer des opérations sur les variables et les valeurs.

  • Opérateurs arithmétiques : comme +, -, *, /, %.
  • Opérateurs de comparaison : comme ==, !=, <, >, <=, >=.
  • Opérateurs d'affectation : comme =, +=, -=, *=, /=.
  • Opérateurs logiques : comme and, or, &&, ||, !.

3.4. Structures de contrôle (if, for, while, switch)

Les structures de contrôle permettent d'exécuter des blocs de code en fonction de certaines conditions ou de répéter un bloc de code plusieurs fois.

  • if : Exécute un bloc de code si une condition spécifiée est vraie.

    <?php
    if ($x > 10) {
    echo "La valeur de x est supérieure à 10.";
    }
    ?>

  • for : Boucle qui exécute un bloc de code un certain nombre de fois.

    <?php
    for ($i = 0; $i < 5; $i++) {
    echo $i;
    }
    ?>

  • while : Boucle qui exécute un bloc de code tant qu'une condition spécifiée est vraie.

    <?php
    while ($x <= 5) {
    echo $x;
    $x++;
    }
    ?>

  • switch : Un type de structure de sélection qui choisit un bloc de code à exécuter parmi plusieurs.

    <?php
    switch ($x) {
    case 1:
    echo "x vaut 1";
    break;
    case 2:
    echo "x vaut 2";
    break;
    default:
    echo "x ne vaut ni 1 ni 2";
    }
    ?>

Partie 4: Fonctions

4.1. Définition et utilisation

En PHP, une fonction est un bloc de code réutilisable qui effectue une action spécifique. Une fois qu'une fonction est définie, elle peut être utilisée (ou "appelée") à plusieurs reprises dans votre code sans avoir à la réécrire.

La structure d'une fonction :

<?php
function nomDeLaFonction() {
// code à exécuter
}
?>

Appeler une fonction :

<?php
nomDeLaFonction();
?>

4.2. Fonctions natives de PHP

PHP fournit une grande variété de fonctions natives qui permettent de réaliser une multitude de tâches sans avoir à écrire le code vous-même. Voici quelques exemples :

  • string strlen(string $string) : Renvoie la longueur d'une chaîne.
  • mixed str_replace(mixed $search, mixed $replace, mixed $subject) : Remplace toutes les occurrences de la recherche par la valeur de remplacement dans la chaîne donnée.
  • int intval(mixed $value) : Renvoie la valeur entière de la variable.
  • bool in_array(mixed $needle, array $haystack) : Vérifie si une valeur existe dans un tableau.

Et bien d'autres! PHP possède une documentation complète qui liste toutes ses fonctions natives et leurs utilisations.

4.3. Portée des variables et global

La portée d'une variable est la partie du script où la variable peut être référencée ou utilisée. En PHP, les variables peuvent être soit locales, soit globales.

  • Variables locales : Une variable déclarée à l'intérieur d'une fonction est locale et ne peut être utilisée que par cette fonction.
  • Variables globales : Une variable déclarée à l'extérieur d'une fonction est globale et peut être utilisée par n'importe quelle fonction. Pour accéder à une variable globale depuis l'intérieur d'une fonction, vous devez utiliser le mot-clé `global`.

<?php
$x = 5; // variable globale
function maFonction() {
global $x;
echo $x;
}
maFonction();
?>

Dans l'exemple ci-dessus, la variable `$x` est déclarée en dehors de la fonction, ce qui en fait une variable globale. Pour l'utiliser à l'intérieur de la fonction `maFonction`, nous devons utiliser le mot-clé `global`.

Partie 5: Gestion de données

5.1. Tableaux (arrays)

Un tableau est une structure de données qui permet de stocker plusieurs valeurs sous un seul nom. Ces valeurs peuvent être de n'importe quel type. En PHP, il existe trois types de tableaux :

  • Indexés : les tableaux avec un indice numérique
  • Associatifs : les tableaux où chaque clé est associée à une valeur
  • Multidimensionnels : les tableaux contenant un ou plusieurs tableaux

<?php
// Tableau indexé
$couleurs = array("rouge", "vert", "bleu");

// Tableau associatif
$ages = array("Jean" => 35, "Marie" => 25, "Anne" => 30);

// Tableau multidimensionnel
$voitures = array(
array("Peugeot", 20, 10),
array("Renault", 30, 20),
array("BMW", 5, 2)
);
?>

5.2. Chaînes de caractères (strings)

En PHP, une chaîne de caractères est une suite de caractères, comme "Bonjour le monde!". Une chaîne peut être n'importe quel texte encadré par des guillemets simples ou doubles.

<?php
$txt1 = "Bonjour le monde!";
$txt2 = 'Bonjour le monde!';
echo $txt1;
echo $txt2;
?>

Il existe de nombreuses fonctions intégrées en PHP pour manipuler des chaînes, comme `strlen()`, `str_replace()`, `substr()`, et bien d'autres.

5.3. Dates et temps

PHP fournit de nombreuses fonctions pour manipuler la date et l'heure. Pour obtenir la date et l'heure actuelles en PHP, vous pouvez utiliser la fonction `date()`.

<?php
echo date("Y-m-d H:i:s");
?>

Avec la fonction `date()`, vous pouvez récupérer différents formats de date et heure en utilisant diverses chaînes de format, comme "Y" pour l'année, "m" pour le mois, "d" pour le jour, "H" pour l'heure, "i" pour les minutes et "s" pour les secondes.

Il est également possible d'obtenir et de manipuler des informations sur le fuseau horaire, d'utiliser des timestamps Unix, et bien plus encore grâce aux fonctions intégrées de PHP liées à la date et l'heure.

Partie 6: Programmation orientée objet (POO) en PHP

6.1. Introduction à la POO

La Programmation Orientée Objet (POO) est un paradigme de programmation qui repose sur le concept d'encapsulation des données et des méthodes qui opèrent sur ces données dans des entités appelées "objets". Elle permet de structurer le code d'une manière plus modulaire et réutilisable. En PHP, la POO a été introduite pour offrir aux développeurs un moyen d'organiser leur code de manière plus efficace et cohérente.

6.2. Classes et objets

Les classes sont des définitions ou des modèles à partir desquels on crée des objets. Un objet est une instance d'une classe.

<?php
class Voiture {
public $couleur;
function afficherCouleur() {
return $this->couleur;
}
}
$maVoiture = new Voiture();
$maVoiture->couleur = "rouge";
echo $maVoiture->afficherCouleur();
?>

6.3. Héritage

L'héritage est un mécanisme qui permet à une classe de hériter des propriétés et des méthodes d'une autre classe. La classe qui hérite est appelée "classe enfant" et la classe dont on hérite est appelée "classe parente".

<?php
class Vehicule {
public $vitesse;
}
class Voiture extends Vehicule {
public $nombreDePortes;
}
?>

6.4. Encapsulation

L'encapsulation est le principe qui permet de cacher les détails de mise en œuvre d'un objet. En PHP, on utilise les modificateurs d'accès (public, private, protected) pour contrôler l'accès aux propriétés et méthodes d'une classe.

6.5. Polymorphisme

Le polymorphisme permet à des objets de différentes classes d'être traités comme s'ils étaient des objets d'une même classe. En PHP, le polymorphisme est principalement réalisé à l'aide d'interfaces et de méthodes abstraites.

6.6. Espaces de noms (namespaces)

Les espaces de noms permettent d'encapsuler des éléments de code, évitant ainsi les conflits de noms entre les classes, fonctions ou constantes. C'est particulièrement utile dans les gros projets ou lors de l'utilisation de bibliothèques tierces.

<?php
namespace MonEspaceDeNoms;
class MaClasse {}
$objet = new MaClasse();
?>

On peut ensuite faire référence à cette classe en utilisant `MonEspaceDeNoms\MaClasse` dans d'autres parties du code.

Partie 7: Gestion des erreurs

7.1. Manipulation des exceptions

Les exceptions sont des événements anormaux qui se produisent pendant l'exécution d'un programme. En PHP, les exceptions sont des objets de la classe `Exception` ou de ses classes dérivées. Elles sont levées (ou "thrown") lorsqu'une erreur se produit et peuvent être "attrapées" (ou "caught") pour permettre la gestion personnalisée de l'erreur.

<?php
try {
throw new Exception("Une erreur s'est produite.");
} catch (Exception $e) {
echo "Exception capturée: " . $e->getMessage();
}
?>

Dans cet exemple, une exception est levée avec un message, puis elle est immédiatement attrapée et le message est affiché.

7.2. Rapports d'erreur et débogage

PHP dispose de plusieurs niveaux de rapports d'erreurs qui peuvent être configurés pour afficher ou masquer certains types d'erreurs. Le réglage des niveaux de rapport d'erreur est crucial lors du développement pour repérer et corriger les bugs, mais il est tout aussi essentiel de limiter les rapports d'erreur en production pour des raisons de sécurité.

<?php
ini_set('display_errors', 1);
ini_set('display_startup_errors', 1);
error_reporting(E_ALL);
?>

Ces instructions vont activer l'affichage de toutes les erreurs. Pour le débogage, PHP offre également des fonctions telles que `var_dump()` et `print_r()` pour afficher les détails des variables, ainsi que des outils tels que Xdebug pour une analyse plus approfondie.

Partie 8: Gestion des formulaires et données d'entrée

8.1. Récupérer des données via GET et POST

En PHP, les données envoyées par un formulaire peuvent être récupérées à l'aide des superglobales `$_GET` et `$_POST`, selon la méthode choisie lors de l'envoi du formulaire. `$_GET` récupère les données transmises via l'URL, tandis que `$_POST` récupère les données transmises via le corps de la requête HTTP.

<form action="traitement.php" method="post">
Nom: <input type="text" name="nom">
<input type="submit">
</form>

Dans "traitement.php", pour récupérer la valeur du champ "nom" :

<?php
$nom = $_POST['nom'];
echo "Bonjour, " . $nom;
?>

8.2. Sécurité des entrées

Il est essentiel de ne jamais faire confiance aux données entrées par l'utilisateur. Ces données doivent toujours être validées et nettoyées avant d'être utilisées, pour prévenir les attaques comme les injections SQL ou les scripts malveillants.

PHP propose des fonctions comme `filter_input()` et `htmlspecialchars()` pour aider à sécuriser les entrées.

Partie 9: Accès à la base de données avec PHP

9.1. Introduction à MySQL et PDO

MySQL est un système de gestion de bases de données relationnelles. Pour interagir avec MySQL à l'aide de PHP, on utilise souvent une interface appelée PDO (PHP Data Objects). PDO fournit une couche d'abstraction de base de données, ce qui signifie que vous pouvez utiliser le même code pour se connecter et extraire des données de différentes bases de données, comme MySQL, PostgreSQL, et SQLite.

9.2. CRUD (Create, Read, Update, Delete)

CRUD est l'acronyme des quatre opérations fondamentales pour la manipulation des données. Voici un bref aperçu :

  • Create : Ajouter de nouvelles données.
  • Read : Lire des données existantes.
  • Update : Mettre à jour des données existantes.
  • Delete : Supprimer des données.

Avec PDO, ces opérations sont généralement effectuées à l'aide de requêtes préparées pour assurer la sécurité.

9.3. Prévention des injections SQL

L'injection SQL est une technique d'attaque où un attaquant peut insérer ou "injecter" du code SQL indésirable dans une requête. Pour prévenir ces attaques, il est recommandé :

  • d'utiliser des requêtes préparées avec PDO ou MySQLi.
  • de ne jamais insérer directement des données utilisateur dans vos requêtes sans les valider ou les nettoyer.
  • d'utiliser les fonctions spécifiques à PHP pour échapper aux caractères potentiellement dangereux.

L'utilisation de requêtes préparées avec PDO est l'une des méthodes les plus efficaces pour se protéger contre les injections SQL.

Partie 10: Sessions et cookies

10.1. Gestion des sessions

Les sessions permettent de stocker des informations sur le serveur pour un utilisateur donné pendant la durée de sa visite. Les sessions sont un moyen de conserver des données entre les différentes requêtes de la même utilisateur. Elles débutent par un appel à la fonction `session_start()` :

<?php
session_start();
$_SESSION['nom'] = "Dupont";
?>

Les données stockées dans `$_SESSION` sont accessibles depuis n'importe quelle page de votre site, tant que `session_start()` est appelé au début de chaque page.

10.2. Utilisation des cookies

Un cookie est une petite quantité de données stockée sur le navigateur de l'utilisateur et renvoyée au serveur avec chaque requête. Les cookies sont souvent utilisés pour identifier un utilisateur ou pour stocker des préférences spécifiques à un utilisateur. Voici comment définir un cookie en PHP :

<?php
setcookie("utilisateur", "Dupont", time()+3600); // Expire après 1 heure
?>

Le cookie ci-dessus stocke le nom "Dupont" et expire après une heure. Vous pouvez récupérer la valeur du cookie avec `$_COOKIE["utilisateur"]`.

Partie 11: Sécurité en PHP

11.1. Techniques d'authentification

L'authentification est le processus qui permet de vérifier l'identité d'un utilisateur. En PHP, cela est souvent réalisé grâce à un système de connexion avec un nom d'utilisateur et un mot de passe. Les sessions sont souvent utilisées pour maintenir l'état authentifié d'un utilisateur après sa connexion.

11.2. Cryptage et sécurité des mots de passe

Il est crucial de ne jamais stocker les mots de passe en clair dans votre base de données. PHP fournit des fonctions pour hasher les mots de passe de manière sécurisée. La fonction `password_hash()` crée un hash de mot de passe, et `password_verify()` est utilisée pour vérifier un mot de passe contre un hash.

<?php
$mot_de_passe = "monPass123";
$hash = password_hash($mot_de_passe, PASSWORD_DEFAULT);
?>

Lors de la vérification du mot de passe, utilisez :

<?php
if (password_verify($mot_de_passe, $hash)) {
echo "Mot de passe valide!";
} else {
echo "Mot de passe invalide!";
}
?>

11.3. Sécurisation des applications PHP

Sécuriser une application PHP va bien au-delà du simple cryptage des mots de passe. Voici quelques meilleures pratiques :

  • Utilisez des requêtes préparées pour éviter les injections SQL.
  • Ne divulguez jamais d'informations sensibles dans vos messages d'erreur.
  • Mettez régulièrement à jour votre version PHP et vos bibliothèques pour éviter les vulnérabilités connues.
  • Utilisez des mesures de sécurité côté client, comme les Content Security Policies (CSP).
  • Évitez d'utiliser des fonctions PHP obsolètes ou déconseillées.

Ces techniques, combinées à une surveillance constante et à des tests de sécurité, aideront à garder votre application PHP sécurisée.

Partie 12: Bonnes pratiques et design patterns

12.1. MVC (Modèle-Vue-Contrôleur)

Le pattern MVC est une architecture qui divise l'application en trois composants interactifs:

  • Modèle: Représente la logique de données et les règles métier. Il interagit directement avec la base de données.
  • Vue: Affiche les données à l'utilisateur. Il représente l'interface utilisateur.
  • Contrôleur: Gère les requêtes utilisateur et met à jour le modèle ou la vue en conséquence.

L'adoption du pattern MVC facilite la modularité, la scalabilité et la maintenance de l'application.

12.2. Singleton, Factory et autres motifs courants

Les design patterns sont des solutions éprouvées à des problèmes courants de conception. Voici quelques-uns des plus courants en PHP :

  • Singleton: Garantit qu'une classe n'a qu'une seule instance et fournit un point d'accès global à cette instance.
  • Factory: Utilisé pour créer des objets sans spécifier la classe exacte à instancier.
  • Observer: Un objet maintient une liste de ses dépendants et les notifie de tout changement d'état.
  • Strategy: Permet à une famille d'algorithmes d'être interchangeable et choisi à la volée lors de l'exécution.

Partie 13: Outils et extensions

13.1. Composer et gestion des dépendances

Composer est un outil de gestion des dépendances pour PHP. Il vous permet de déclarer les bibliothèques dont dépend votre projet et il les gèrera (installera/mettra à jour) pour vous. Pour l'utiliser :

composer require [package-name]

Composer crée un fichier `composer.json` qui liste toutes les dépendances de votre projet.

13.2. Frameworks populaires (Laravel, Symfony)

Les frameworks PHP facilitent le développement en fournissant une structure et des fonctions préconstruites :

15.1. Mise en production

La mise en production d'une application PHP nécessite une planification soignée. Il est essentiel de :

  • Configurer correctement le serveur (Apache, Nginx).

15.2. Optimisation de la performance

Pour assurer une expérience utilisateur fluide, il est crucial d'optimiser la performance de votre application PHP :

  • Utiliser un optimiseur PHP comme OpCache.
  • Optimiser les requêtes SQL.
  • Minimiser et compresser les ressources (CSS, JS).
  • Utiliser la mise en cache côté serveur et côté client.

15.3. Cache et accélération

Le caching est une technique qui stocke des copies temporaires de contenu pour réduire la charge sur le serveur et améliorer la vitesse de chargement :

  • Cache d'opcode: Stocke le code compilé pour éviter de recompiler à chaque requête.
  • Cache d'objet: Stocke des objets ou des données pour réduire le nombre de requêtes à une base de données ou à une API.
  • Cache de page: Stocke la sortie HTML complète d'une page pour éviter le rendu répété.

Des outils comme Memcached ou Redis sont souvent utilisés pour faciliter la mise en cache en PHP.
